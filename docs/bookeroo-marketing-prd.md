# Product Requirements Document (PRD)
## Bookeroo Marketing Website

# 1. Product Overview

Product Name: Bookeroo
Product Type: SaaS platform for tour operators and activity businesses

The Bookeroo marketing website exists to:

• attract tour operators
• explain the platform clearly
• demonstrate product functionality
• build trust with operators
• generate demo bookings

Primary conversion goal:

Book a Demo

The website must communicate that Bookeroo is the easiest platform to run a tour business.

Core value proposition:

Run your entire tour business in one place.

Supporting value points:

• extremely easy to use
• modern interface
• powerful staffing tools
• operational visibility
• tailored onboarding support
• simple setup

# 2. Target Customers

Primary early focus:

Adventure operators

Examples:

• boat tours
• jet ski rentals
• ATV tours
• parasailing operators
• charter companies

Secondary markets:

• escape rooms
• walking tours
• food tours
• activity providers
• guided experiences

The messaging should speak to **tour operators**, not customers booking tours.

# 14. Component System

The marketing site must be built using a reusable component architecture to ensure consistency, scalability, and maintainability.

Core components should include:

• Navbar
• Footer
• Hero section
• Feature grid
• Product screenshot panel
• CTA section
• Pricing table
• FAQ accordion
• Industry page template
• Blog post template
• Product feature page template

Component goals:

• maximize reuse across pages
• maintain consistent design language
• support rapid creation of SEO pages
• simplify future expansion of the site

All pages should be constructed using these reusable components rather than custom one-off layouts.

# 15. Design Consistency Rules

The entire website must follow a consistent visual system.

All pages should share the same design language, including:

• consistent spacing and layout grid
• consistent card styles
• consistent gradient usage
• consistent typography scale
• consistent CTA button styling
• consistent screenshot framing
• consistent animation style

Design goals:

• premium SaaS aesthetic
• modern interface
• clear visual hierarchy
• high readability
• strong conversion focus

The site should feel cohesive across all pages and sections.
