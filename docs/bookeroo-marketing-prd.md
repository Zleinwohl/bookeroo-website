# Product Requirements Document (PRD)
## Bookeroo Marketing Website

# 1. Product Overview

Product Name: Bookeroo
Product Type: SaaS platform for tour operators and activity businesses

The Bookeroo marketing website exists to:

• attract tour operators
• explain the platform clearly
• demonstrate product functionality
• build trust with operators
• generate demo bookings

Primary conversion goal:

Book a Demo

The website must communicate that Bookeroo is the easiest platform to run a tour business.

Core value proposition:

Run your entire tour business in one place.

Supporting value points:

• extremely easy to use
• modern interface
• powerful staffing tools
• operational visibility
• tailored onboarding support
• simple setup

# 2. Target Customers

Primary early focus:

Adventure operators

Examples:

• boat tours
• jet ski rentals
• ATV tours
• parasailing operators
• charter companies

Secondary markets:

• escape rooms
• walking tours
• food tours
• activity providers
• guided experiences

The messaging should speak to **tour operators**, not customers booking tours.

# 3. Website Objectives

Explain the product clearly
Demonstrate the dashboard
Drive demo bookings
